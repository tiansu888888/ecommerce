Knowledge Check Practice 知识自测练习

1. Which security dimension ensures that data is NOT altered during transmission? 哪个安全维度确保数据在传输过程中不被篡改？

Explanation: Integrity ensures that data is unaltered during transit from sender to receiver. 解析：完整性确保数据在从发送方到接收方的传输过程中不被篡改。

2. Non-repudiation in e-commerce security prevents which of the following? 电子商务安全中的不可否认性可防止以下哪种情况？

Explanation: Non-repudiation prevents a user from denying they placed an order or made a transaction. 解析：不可否认性可防止用户否认他们下过订单或进行过交易。

3. Which security threat redirects users to spoofed fake websites by hijacking or poisoning DNS records? 哪种安全威胁通过劫持或毒化 DNS 记录将用户重定向到虚假网站？

Explanation: Pharming redirects traffic to fake websites by poisoning DNS servers (manipulating hostname IP mappings). 解析：域名劫持（Pharming）通过毒化 DNS 服务器（篡改域名 IP 映射）将流量重定向到假网站。

4. A symmetric key cryptography system uses how many keys? 对称密钥密码系统使用多少个密钥？

Explanation: Symmetric key cryptography uses a single key shared between sender and receiver for both encryption and decryption. 解析：对称密钥密码学使用发送方和接收方之间共享的单个密钥进行加密和解密。

5. An asymmetric (public-key) cryptography system uses __________________. 非对称（公钥）密码系统使用 __________________。

Explanation: Asymmetric encryption utilizes a public key (shared globally) and a private key (kept secret by the owner). 解析：非对称加密使用公钥（全局共享）和私钥（由所有者保密）。

6. What is the function of a Digital Certificate in SSL/TLS? SSL/TLS 中的数字证书有什么功能？

Explanation: A digital certificate issued by a CA validates website identity and supplies the public key for SSL/TLS connections. 解析：CA 签发的数字证书可验证网站身份并为 SSL/TLS 连接提供公钥。

7. Which of the following is a network security tool designed to filter incoming traffic based on predefined rules? 以下哪项是旨在根据预定义规则过滤传入流量的网络安全工具？

Explanation: A firewall filters inbound and outbound network packet traffic based on access rules. 解析：防火墙根据访问规则过滤进出网络数据包的流量。

8. Which security threat inserts malicious database queries into website input forms to steal records? 哪种安全威胁将恶意数据库查询插入网站输入表单以窃取记录？

Explanation: SQL Injection inserts malicious SQL command syntax into vulnerable web input forms to hijack backend databases. 解析：SQL 注入将恶意 SQL 命令语法插入有漏洞的 Web 输入表单中以劫持后端数据库。

Revision Review Q&A 课程复习问答

Review Exercise 1 复习练习题 1 ▶

Describe the function of a Digital Certificate and outline the step-by-step process of how a website obtains one. 描述数字证书（Digital Certificate）的功能，并概述网站如何获取证书的分步过程。

1. Function (功能): A digital document issued by a trusted third party (Certificate Authority) (1 Mark) that authenticates the website's identity to browsers and enables SSL/TLS encryption (1 Mark).
2. Obtaining Process (获取过程):
Step 1: Website owner generates a private key and a Certificate Signing Request (CSR) (1 Mark).
Step 2: Owner submits the CSR along with proof of organization identity to a trusted Certificate Authority (1 Mark).
Step 3: The CA validates the owner's identity and organization ownership.
Step 4: Once confirmed, the CA signs and issues the digital certificate containing the owner's public key (1 Mark). 1. 功能：由受信任第三方（证书颁发机构 CA）签发的电子文档（1分），用于向浏览器认证网站身份并启用 SSL/TLS 加密（1分）。
2. 获取过程：
步骤 1：网站所有者生成私钥和证书签名请求 (CSR)（1分）。
步骤 2：所有者向受信任的证书颁发机构提交 CSR 以及组织身份证明（1分）。
步骤 3：CA 验证所有者的身份和组织所有权。
步骤 4：确认后，CA 签发并颁发包含所有者公钥的数字证书（1分）。
Review Exercise 2 复习练习题 2 ▶

Explain how Public Key Cryptography systems protect data confidentiality during transit. Contrast Symmetric and Asymmetric systems. 解释公钥密码系统如何保护传输中的数据机密性。对比对称和非对称系统。

1. Confidentiality Protection (机密性保护): Asymmetric cryptography uses a mathematically related Public Key and Private Key (1 Mark). The sender encrypts plaintext data using the receiver's public key. Once encrypted, the ciphertext can ONLY be decrypted using the receiver's private key (1 Mark). Even if intercepted, the message remains unreadable (confidential) without the secret private key.
2. Contrast (对比):
Symmetric encryption uses a single shared key for both encrypting and decrypting bulk data (1 Mark). It is fast but has key distribution vulnerability. Asymmetric encryption uses two keys, solving the key distribution problem, but is computationally slow (1 Mark). 1. 机密性保护：非对称密码学使用数学相关的公钥和私钥（1分）。发送方使用接收方的公钥对明文数据进行加密。加密后，密文只能使用接收方的私钥进行解密（1分）。即使被拦截，在没有保密私钥的情况下，消息仍无法阅读（保持机密）。
2. 对比：
对称加密使用单个共享密钥来加密和解密批量数据（1分）。它速度快，但存在密钥分发漏洞。非对称加密使用两个密钥，解决了密钥分发问题，但计算慢（1分）。
