Knowledge Check Practice 知识自测练习

1. What is the primary technical concept behind internet packet switching? 互联网分组交换背后的主要技术概念是什么？

Explanation: Packet switching slices digital messages into packets, routes them dynamically along different paths, and reassembles them at the destination. 解析：分组交换将数字消息切成数据包，沿不同路径动态路由，并在目的地重新组装。

2. The Domain Name System (DNS) translates __________________. 域名系统（DNS）翻译 __________________。

Explanation: DNS is a hierarchical database that translates human-readable domain names (e.g. www.google.com) into IP addresses. 解析：DNS 是一个分层数据库，用于将人类可读的域名（例如 www.google.com）翻译为 IP 地址。

3. Why is HTTPS critical for securing e-commerce transactions compared to HTTP? 与 HTTP 相比，为什么 HTTPS 对保障电子商务交易安全至关重要？

Explanation: HTTPS encrypts the data transport layer via SSL/TLS protocols, protecting sensitive user data from eavesdropping. 解析：HTTPS 通过 SSL/TLS 协议对数据传输层进行加密，保护敏感的用户数据免受窃听。

4. IPv4 (Internet Protocol version 4) addresses utilize how many bits? IPv4（互联网协议第 4 版）地址使用多少位？

Explanation: IPv4 addresses use a 32-bit structure represented as four decimal bytes separated by periods. 解析：IPv4 地址采用 32 位结构，表示为由点分隔的四个十进制字节。

5. Which protocol is used to transfer web pages from servers to client browsers? 哪种协议用于将网页从服务器传输到客户端浏览器？

Explanation: HTTP (Hypertext Transfer Protocol) is the protocol used to request and transmit web content. 解析：HTTP（超文本传输协议）是用于请求和传输网页内容的协议。

6. Which layer of the TCP/IP protocol suite is responsible for routing packets across networks? TCP/IP 协议族的哪一层负责跨网络路由数据包？

Explanation: The Internet layer of TCP/IP (such as the IP protocol) handles routing of data packets across network boundaries. 解析：TCP/IP 的网际网络层（例如 IP 协议）处理跨网络边界的数据包路由。

7. The Web runs on top of which infrastructure? Web 运行在哪个基础设施之上？

Explanation: The Web is an information service that runs on top of the physical network infrastructure of the Internet. 解析：Web 是一种信息服务，运行在互联网的物理网络基础设施之上。

8. What does latency refer to in network communication? 网络通信中的时延是指什么？

Explanation: Latency is the delay or round-trip time it takes for a packet of data to travel from source to destination. 解析：时延是数据包从源传输到目的地所花费的延迟或往返时间。

Revision Review Q&A 课程复习问答

Review Exercise 1 复习练习题 1 ▶

Describe the Packet Switching concept and explain why it is more efficient than circuit-switched networks. 描述分组交换（Packet Switching）概念，并解释为什么它比电路交换网络更高效。

1. Packet Switching Concept (分组交换概念): Slices digital messages into packets (1 Mark), sends them along different communication paths (1 Mark), and reassembles them at the destination (1 Mark).
2. Efficiency (高效性): It avoids dedicated circuits. It allows multiple users to share lines (1 Mark), dynamically routes around busy nodes (1 Mark), and maximizes bandwidth utilization (1 Mark) by only consuming capacity when data is actively sent. 1. 分组交换概念：将数字消息切成数据包（1分），沿不同的通信路径发送它们（1分），并在目的地重新组装它们（1分）。
2. 高效性：它避免了专用电路。它允许多个用户共享线路（1分），动态绕过繁忙节点路由（1分），并且仅在主动发送数据时才消耗容量，从而最大化带宽利用率（1分）。
Review Exercise 2 复习练习题 2 ▶

Explain the Domain Name System (DNS) functions, specifically domain resolution and database management. 解释域名系统（DNS）的功能，特别是域名解析和数据库管理。

1. Domain Resolution (域名解析): DNS translates human-friendly domain URLs (e.g. www.google.com) into numeric IP addresses (e.g. 172.217.7.14) that routers need to forward packets.
2. Distributed Database Management (分布式数据库管理): DNS operates as a hierarchical, distributed database system. No single server holds all mappings; instead, root, TLD, and authoritative name servers coordinate globally to manage and resolve addresses. 1. 域名解析：DNS 将人类易记的域名 URL（例如 www.google.com）翻译成路由器转发数据包所需的数字 IP 地址（例如 172.217.7.14）。
2. 分布式数据库管理：DNS 作为一个分层的分布式数据库系统运行。没有单个服务器保存所有映射；相反，根、TLD 和权威域名服务器在全球范围内协调以管理和解析地址。
