Knowledge Check Practice 知识自测练习

1. Which phase of the Systems Development Life Cycle (SDLC) defines the system's business objectives and requirements? 系统开发生命周期（SDLC）的哪个阶段定义了系统的业务目标和需求？

Explanation: Systems Analysis/Planning is the first phase of SDLC, where business goals are identified and translated into requirements. 解析：系统分析/规划是 SDLC 的第一阶段，在此阶段识别业务目标并将其转化为需求。

2. A logical system design describes __________________. 逻辑系统设计描述了 __________________。

Explanation: Logical design defines what the system should do conceptually (specifications, data flow) without specifying actual physical hardware. 解析：逻辑设计定义了系统在概念上应该做什么（规范、数据流），而不指定实际的物理硬件。

3. What is unit testing in e-commerce site development? 电子商务网站开发中的单元测试是什么？

Explanation: Unit testing involves testing specific modules of code, forms, or pages in isolation to ensure they perform correctly before integration. 解析：单元测试涉及隔离测试特定的代码模块、表单或页面，以确保它们在集成前正确运行。

4. Which of the following is a major advantage of outsourcing website development? 以下哪项是外包网站开发的主要优势？

Explanation: Outsourcing leverages the vendor's specialized experience, lowering development risk and accelerating deployment. 解析：外包利用了开发商的专业经验，降低了开发风险并加速了部署。

5. A physical system design specifies which of the following? 物理系统设计指定了以下哪项？

Explanation: Physical design translates the logical design into actual physical components, naming specific hardware, databases, and software languages. 解析：物理设计将逻辑设计转化为实际的物理组件，指明具体的硬件、数据库和软件语言。

6. What does SDLC stand for? SDLC 代表什么？

Explanation: SDLC stands for Systems Development Life Cycle, a methodology for managing the planning, building, testing, and deployment of systems. 解析：SDLC 代表系统开发生命周期，是管理系统规划、构建、测试和部署的方法论。

7. System testing checks __________________. 系统测试检查 __________________。

Explanation: System testing tests the entire integrated website, confirming that shopping carts, payment gateways, and inventories cooperate correctly. 解析：系统测试对整个集成网站进行测试，确认购物车、支付网关和库存协同工作正常。

8. Which practice compares your site's operational metrics against industry top performers? 哪项实践将您网站的运营指标与行业顶尖企业进行对比？

Explanation: Benchmarking compares a firm's system metrics (loading speed, transaction rates) against industry standards or leading competitors. 解析：基准测试将公司的系统指标（加载速度、交易率）与行业标准或领先的竞争对手进行比较。

Revision Review Q&A 课程复习问答

Review Exercise 1 复习练习题 1 ▶

Describe the System Development Life Cycle (SDLC) phases specifically for developing an e-commerce website. 描述开发电子商务网站特定的系统开发生命周期（SDLC）阶段。

The SDLC methodology consists of 5 main phases:
1. Systems Analysis (系统分析): Identify business goals and translate them into system functionalities.
2. Systems Design (系统设计): Map out system specifications, including logical (data flow) and physical (actual servers, languages) designs.
3. Building the System (构建系统): Code, compile, configure servers, and build templates.
4. Testing (测试): Execute unit tests on isolated modules and system tests on the integrated site to ensure no bugs remain.
5. Implementation & Maintenance (实施与维护): Launch the website live, monitor performance, and continuously update database systems. SDLC 方法论由 5 个主要阶段组成：
1. 系统分析：确定业务目标并将其转化为系统功能。
2. 系统设计：规划系统规范，包括逻辑（数据流）和物理（实际服务器、语言）设计。
3. 构建系统：编写代码、编译、配置服务器并构建模板。
4. 测试：对独立模块执行单元测试，对集成网站执行系统测试，确保无漏洞。
5. 实施与维护：使网站上线，监控性能，并持续更新数据库系统。
Review Exercise 2 复习练习题 2 ▶

Differentiate between System Logical Design and System Physical Design, explaining why both are necessary. 区分系统逻辑设计与系统物理设计，并解释为什么两者都是必要的。

1. System Logical Design (系统逻辑设计): Focuses on data flows, input/output requirements, and processing rules conceptually without specifying hardware brands or code syntax (1 Mark). It ensures business specifications are logically complete.
2. System Physical Design (系统物理设计): Translates logical models into concrete physical layouts, specifying database software (e.g. MySQL), server architectures (e.g. CPU, RAM), and coding languages (e.g. PHP/Python) (1 Mark).
Necessity (必要性): Logical design ensures the system meets user business needs. Physical design provides the exact build plan to guide programmers and hardware purchases. 1. 系统逻辑设计：侧重于数据流、输入/输出要求和处理规则的概念，不指定硬件品牌或代码语法（1分）。它确保业务规范在逻辑上完整。
2. 系统物理设计：将逻辑模型转化为具体的物理布局，指定数据库软件（如 MySQL）、服务器架构（如 CPU、RAM）和编程语言（如 PHP/Python）（1分）。
必要性：逻辑设计确保系统满足用户的业务需求。物理设计提供了确切的构建计划，以指导程序员和硬件采购。
